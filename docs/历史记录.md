## 历史记录
在创建请求的时候，把请求的相关参数存入到本地 再把一些结果的一些参数也存储到本地形成一个完整的数据 然后渲染到我们的历史列表里面
有哪些需要收集的参数列在这文档中 ：

具体需求 ：
1. 从创建请求开始收集 只到结束 每次状态呀改变的时候 需要更新到本地 
2. 每次打开历史界面 调用/tts/details获取任务详情  更新本地数据
3. 如果状态为失败 历史记录表里面需要用一个重试按钮 ； 如果为成功 需要一个下载字幕 复制文本按钮
4. 目前的布局不合适 历史界面太小  可以中间弹窗一个table表的形式 来显示数据 
### 存储结构示例 （参数名按可按返回参数名）
```typescript
interface TaskHistoryItem {
  id: string;                // 任务唯一标识
  videoSource: string;       // 视频来源
  fileName?: string;         // 文件名（如果是上传）
  fileSize?: number;         // 文件大小
  language: string;          // 选择的语言
  timestamp: Date;           // 创建时间
  status: 'queued' | 'processing' | 'completed' | 'failed'; // 状态
  completedAt?: Date;        // 完成时间
  processingTime?: number;   // 处理耗时
  downloadUrl?: string;      // 下载链接
  errorInfo?: string;        // 错误信息
}
```
